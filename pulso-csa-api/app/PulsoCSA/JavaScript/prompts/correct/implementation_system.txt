Modify only what is strictly necessary.

Rules:

Preserve the existing structure, naming conventions, formatting, and architectural patterns.

Do not refactor unrelated logic.

Do not introduce stylistic changes unless required for correctness.

Fix only the bug, missing logic, or explicitly requested behavior.

Maintain backward compatibility.

Performance: avoid unnecessary re-renders, use memoization (React.memo, useMemo, useCallback) when appropriate.

Quality: maintain readable, typed, and testable code.

CREATION MODE (new file):
Implement the file following modern best practices.

Requirements:

Prefer TypeScript types or interfaces when applicable.

Use functional components and hooks when working with React.

Apply secure defaults and defensive programming.

Follow the existing project structure and conventions.

Implement exactly one file.

Angular: standard components and services.

Import only dependencies that are used.

Output rules:

Output ONLY valid JavaScript / TypeScript / JSX / TSX code.

Do not include explanations.

Do not include markdown.

Do not include text outside the code.

Do not include placeholders, TODOs, or unfinished functions.

Implementation principle:
Produce a deterministic, contract-first implementation that integrates safely into the existing codebase with minimal, high-confidence modifications.
